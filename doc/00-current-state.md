# coffeTide 현재 정본 기획서

> 프로젝트명: **coffeTide** (구 TimePilot) · 서비스 도메인(예정): `coffeTide.dongple.kr` · 기본 플랫폼: **웹** (모바일 전략: [`8-mobile_strategy.md`](./8-mobile_strategy.md))
> 이 문서가 제품의 정본 기획입니다. **2026-07-11 MVP 구현 완료** — 구현 현황은 [`as-built-reference.md`](./as-built-reference.md), 남은 격차는 [`7-backlog.md`](./7-backlog.md) 참조.

## 1. 제품 한 줄 정의

coffeTide는 **커피 한 잔 하면서 오늘을 정리하는 AI 업무 비서** — 연동이 없어도 오늘의 일을 정리하고, 연동되면 자동 수집과 실행력이 강해집니다.

기존 목표가 Outlook/Notion 데이터를 가져와 보여주는 대시보드였다면, 현재 정본 목표는 사용자가 어떤 방식으로든 업무 단서를 넣으면 coffeTide가 이를 구조화하고 우선순위와 다음 행동으로 바꾸는 것입니다.

## 2. 핵심 제품 원칙

1. **무연동 사용 가능**
   - Outlook, Notion, Obsidian, 로컬 문서 폴더가 모두 없어도 사용자는 직접 업무를 입력할 수 있어야 합니다.
   - Copilot은 외부 연동이 없어도 수동 입력 데이터만으로 동작해야 합니다.

2. **수동/로컬 데이터는 1급 소스**
   - `manual`, `paste`, `local_doc`, `obsidian`, `outlook`, `notion`은 모두 같은 업무 데이터 파이프라인으로 들어갑니다.
   - 수동 입력 데이터도 행동 지침, 알림, Copilot 브리핑, 완료 처리 대상입니다.

3. **연동은 시작 조건이 아니라 증강 기능**
   - Outlook은 메일 자동 수집과 답장 초안 저장을 강화합니다.
   - Notion은 태스크 자동 수집과 원본 상태 write-back을 강화합니다.
   - Obsidian/로컬 문서 폴더는 사용자가 이미 쌓아둔 문서에서 업무 단서를 자동 추출합니다.

4. **부분 실패 허용**
   - 한 소스가 실패해도 다른 소스의 데이터는 계속 사용합니다.
   - 실패 상태는 사용자를 막는 에러가 아니라 대체 입력 경로를 제안하는 상태입니다.

5. **AI 답변은 날짜와 출처에 근거**
   - Copilot은 현재 날짜와 타임존을 입력 컨텍스트로 받아야 합니다.
   - 날짜를 추정하지 않습니다.
   - 주요 업무 제안에는 근거 출처 파일/메일/페이지명을 함께 표시합니다.

## 3. 정본 데이터 허브

모든 입력은 `UnifiedData` 형태로 정규화합니다.

| Source | 설명 | 필수 여부 | write-back |
| :--- | :--- | :--- | :--- |
| `manual` | 사용자가 직접 추가한 업무 | 기본 | 로컬 완료/삭제 |
| `paste` | 메모/메일/회의록 붙여넣기에서 추출한 업무 | 기본 | 로컬 완료/삭제 |
| `local_doc` | 선택한 로컬 폴더의 `.txt`, `.md` 등 문서에서 추출한 업무 | 선택 | 로컬 상태 관리 |
| `obsidian` | Obsidian Vault 문서에서 추출한 업무 | 선택 | 로컬 상태 관리 |
| `outlook` | Microsoft Graph 메일 | 선택 | 답장 초안 생성 |
| `notion` | Notion DB 태스크 | 선택 | 페이지 상태 완료 처리 |
| `gmail` | Google Gmail·Calendar·Drive | 선택 | 읽기 전용 (답장 초안은 백로그) |
| `llm` | 로컬 LLM 도구 산출물(`MEMORY.md` 등) 폴더 스캔 | 선택 | Obsidian 일일 다이제스트 미러링 ([`phase6_llm_artifacts_spec.md`](./phase6_llm_artifacts_spec.md)) |

## 4. 사용자 흐름 정본

### 4.1 첫 진입

1. 사용자가 `coffeTide 시작하기`를 누릅니다.
2. Microsoft 인증 없이 게스트 세션을 발급합니다.
3. 대시보드에는 다음 입력 경로가 먼저 보여야 합니다.
   - 빠른 업무 추가
   - 메모/회의록 붙여넣기
   - 로컬 문서 폴더 선택
   - Outlook/Notion/Obsidian 연결

### 4.2 연동이 없을 때

1. 사용자는 직접 업무를 추가하거나 메모를 붙여넣습니다.
2. coffeTide는 내용을 분류하고 행동 지침을 생성합니다.
3. Copilot은 수동 데이터만으로도 오늘의 우선순위, 예상 시간, 리스크를 브리핑합니다.
4. 사용자는 로컬 완료/삭제/보류 처리를 할 수 있습니다.

### 4.3 연동이 있을 때

1. 연결된 소스만 선택적으로 동기화합니다.
2. 수동/로컬/외부 데이터를 병합합니다.
3. 중복 가능성이 높은 항목은 병합 후보로 표시합니다.
4. 각 소스별로 가능한 액션을 제공합니다.
   - manual/paste/local_doc/obsidian: 완료, 보류, 삭제, 원문 열기
   - outlook: 답장 초안 생성, 원문 열기
   - notion: 완료 처리, 원문 열기

## 5. 구현 필수 요구사항 (정본 준수 체크리스트)

구현·수정 시 아래를 항상 지켜야 합니다. 2026-07-11 MVP에서 모두 반영되었으며([`7-backlog.md`](./7-backlog.md) G 항목), 이후 변경에서도 회귀하면 안 됩니다.

- `UnifiedData.source`에 `manual`, `paste`, `local_doc`, `obsidian`을 명시합니다. (G1 ✅)
- 외부 연동이 없어도 Copilot 입력창이 활성화되어야 합니다. (G3 ✅)
- 빈 화면은 "서비스를 연결하세요"가 아니라 "업무를 직접 추가하거나 문서를 가져오세요"가 기본 안내가 되어야 합니다. (G2 ✅)
- Copilot 응답은 Markdown 원문 노출 대신 카드/섹션 형태로 렌더링해야 합니다. (G6 ✅)
- Copilot은 현재 날짜와 출처 근거를 명시해야 하며, 임의로 오늘 날짜를 추정하면 안 됩니다. (G4 ✅)

## 6. 성공 기준

- Outlook/Notion 미연동 상태에서도 사용자가 1분 안에 업무 1건을 등록하고 Copilot 브리핑을 받을 수 있습니다.
- 로컬 문서 폴더만 연결해도 오늘 해야 할 행동 지침이 생성됩니다.
- 외부 연동이 실패해도 대시보드는 사용 불가 상태가 되지 않습니다.
- Copilot 답변에는 기준일, 출처, 다음 행동, 리스크가 명확히 들어갑니다.
